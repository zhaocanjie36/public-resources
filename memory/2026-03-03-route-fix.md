## 问题分析

SSH 的 `-L` 参数默认会同时绑定 IPv4 (127.0.0.1) 和 IPv6 (::1)。错误信息显示 IPv6 绑定失败了。

可能的原因：
1. **系统 IPv6 未正确配置**
2. **端口 3128 已被占用**
3. **IPv6 回环地址不可用**

## 解决方案

### 方案 1：只绑定 IPv4（推荐）

```bash
ssh -L 127.0.0.1:3128:localhost:3128 -N root@100.122.14.35
```

明确指定绑定到 `127.0.0.1`，不使用 IPv6。

### 方案 2：检查端口是否被占用

```bash
lsof -i :3128
```

如果有输出，说明端口已被占用，需要：
- 杀掉占用进程
- 或者换个端口，比如 `3129`

### 方案 3：禁用 IPv6 绑定

在 SSH 配置中添加：

```bash
ssh -L 127.0.0.1:3128:localhost:3128 -N -o AddressFamily=inet root@100.122.14.35
```

`-o AddressFamily=inet` 强制只使用 IPv4。
